==========================================
  进程PCB管理系统 - 快速使用指南
==========================================

【启动程序】
运行: cargo run

【基本操作流程】

1. 查看初始化快照
   - 程序启动后自动显示初始化快照
   - 此时运行队列为空，PCB池为满（128个空闲）

2. 创建测试进程
   - 选择菜单选项 9：批量创建测试进程
   - 系统会自动创建5个进程（PID 1-5，优先级1-5）
   - 这些进程会自动加入就绪队列

3. 执行调度
   - 选择菜单选项 6：执行一次调度周期
   - 系统会从就绪队列选择一个进程运行
   - 运行进程的时间片会减少

4. 查看系统状态
   - 选择菜单选项 7：显示系统状态
   - 可以看到所有队列的详细信息

5. 测试进程原语
   - 选择 4：挂起进程（输入PID，如1）
   - 选择 5：激活进程（输入PID，如1）
   - 选择 3：时间片到（当前运行进程转为就绪）

6. 生成快照
   - 选择 8：生成快照
   - 选择 1：初始化快照
   - 选择 2：运行前快照
   - 选择 3：运行后快照

【实验报告要点】

1. 初始化快照分析
   - 运行队列：空
   - PCB池：满（128个空闲，0个已用）
   - 总链队列：空
   - 就绪队列：空
   - 等待队列：空

2. 运行前快照分析
   - 创建几个进程后生成
   - 运行队列：可能有进程或为空
   - PCB池：非满（部分已用）
   - 总链队列：包含所有已创建进程
   - 就绪队列：包含就绪状态的进程

3. 运行后快照分析
   - 执行调度后生成
   - 观察进程状态变化
   - 观察队列变化
   - 观察PCB池使用情况

【功能测试建议】

✓ 创建进程：测试不同优先级
✓ 撤销进程：测试PCB回收
✓ 时间片到：测试状态转换
✓ 挂起/激活：测试等待队列
✓ 调度周期：测试CPU调度
✓ 快照功能：记录系统状态变化

【注意事项】

- PCB池最大容量：128
- 默认时间片：5个调度周期
- 进程优先级：建议0-10
- 系统会维护队列一致性
